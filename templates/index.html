Алгоритм Диница

Об алгоритме

Этот алгоритм, опубликованный в 1970 г., имел огромное значение. На протяжении 10 лет все достижения в решении задачи о максимальном потоке базировались на методе Диница.

Основные идеи алгоритма Диница включают в себя использование слоистой сети для поиска блокирующего потока и увеличение потока в основной сети на основе найденного блокирующего потока. Алгоритм Диница также использует методы быстрого поиска пути в слоистой сети и оптимизации поиска блокирующего потока, чтобы уменьшить время выполнения алгоритма. Это делает его эффективнее, в сравнении с другими алгоритмами для нахождения максимального потока в транспортной сети.

Задача о нахождении максимального потока

Входные данные

В первой стоке входного файла должно содержится четыре числа: n (2 ≤ n ≤ 109), m (2 ≤ m ≤ 109), s и t. Где n и m — количество вершин и рёбер в графе, в котором вам требуется найти поток, s и t - исток и сток соответственно. Далее следуют описания рёбер графа, по одному в каждой строке входного файла. Описание ребра состоит из трёх чисел: a, b, c (1 ≤ a, b ≤ n, a ≠ b, 1 ≤ c ≤ 100) Эти числа означают, что из a вершины в вершину b идёт ребро пропускной способности c. Граф не должен содержать кратных рёбер и петель.

Выходные данные

В выводе будет получен размер максимальный поток из вершины s в вершину t.

Пример

Входные данные

4 5 1 4
1 2 2
1 3 3
3 2 1
2 4 3
3 4 2

Выходные данные

5
